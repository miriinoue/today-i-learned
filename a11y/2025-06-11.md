# 達成基準3.3.2: ラベルまたは説明（A）と関連基準の違いまとめ

## ポイント

- **3.3.2 ラベルまたは説明（A）**
  - フォーム要素に視覚的にラベルや説明が提供されているか（見た目重視）
  - コーディングでの関連付けは問わない

- **1.3.1 情報および関係性（A）**
  - ラベルがプログラム的に正しく関連付けられているか
  - 例：`<label>`と`<input>`の関連付け
  - 支援技術が理解できる構造であること

- **4.1.2 名前・役割・値（A）**
  - 支援技術がアクセシブルな名前を取得できるか
  - `aria-label`や`aria-labelledby`等の代替手段も含む

## 具体例

- ❌ 3.3.2失敗 / 1.3.1失敗

  ```html
  <input type="text" name="email">
  <!-- 視覚的なラベルなし、プログラム的関連付けもなし -->
  ```

- ⚠️ 3.3.2合格 / 1.3.1失敗

  ```html
  <span>メールアドレス</span>
  <input type="text" name="email">
  <!-- 視覚的にはラベルあり、でもプログラム的関連付けなし -->
  ```

- ⚠️ 3.3.2失敗 / 1.3.1合格 / 4.1.2合格

  ```html
  <input type="text" name="email" aria-label="メールアドレス">
  <!-- プログラム的にはOK、でも視覚的なラベルが見えない -->
  ```

- ✅ すべて合格

  ```html
  <label for="email">メールアドレス</label>
  <input type="text" id="email" name="email">
  <!-- 視覚的にもプログラム的にも適切 -->
  ```

## 補足

- 3.3.2は「見た目」の話、1.3.1と4.1.2は「技術的実装」の話。
- 3.3.2では、ラベルや説明が正確にマークアップされているかどうかは問わない。
- コントロールが適切なアクセシブルな名前を持っていても、視覚的なラベルや説明がなければ3.3.2は不合格となる場合がある。
